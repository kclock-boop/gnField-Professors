# 관리자 운영 매뉴얼

대한민국산업현장교수 경남협의회 시험용 홈페이지의 간단 로그인 운영 문서입니다.

## 1. 현재 계정 구조

### 관리자 계정

| 구분 | 아이디 | 비밀번호 |
|---|---|---|
| 관리자 | `admin` | `admin1234` |

### 교수님 계정 예시

| 성함 | 아이디 | 초기 비밀번호 |
|---|---|---|
| 김교수 | `gn01` | `1001` |
| 이교수 | `gn02` | `1002` |
| 박교수 | `gn03` | `1003` |
| 최교수 | `gn04` | `1004` |
| 정교수 | `gn05` | `1005` |
| 강교수 | `gn06` | `1006` |
| 조교수 | `gn07` | `1007` |
| 윤교수 | `gn08` | `1008` |
| 장교수 | `gn09` | `1009` |
| 임교수 | `gn10` | `1010` |

성함은 실제 교수님 성함에 맞게 바꿔 사용하시면 됩니다.

## 2. 비밀번호 변경 방식

이제는 로그인 후 상단의 `비밀번호 변경` 버튼으로 각 계정이 직접 바꿀 수 있습니다.

- 교수님은 본인 아이디로 로그인
- `비밀번호 변경` 클릭
- 현재 비밀번호 입력
- 새 비밀번호 입력 후 저장

참고:
- 변경된 비밀번호는 해당 브라우저의 `localStorage`에 저장됩니다.
- 같은 컴퓨터, 같은 브라우저에서는 유지됩니다.
- 다른 컴퓨터로 바꾸면 다시 초기 비밀번호 상태처럼 보일 수 있습니다.

## 3. 코드에서 계정 바꾸는 위치

계정 기본값은 `js/main.js`에 들어 있습니다.

수정 위치 예시:

```js
{ id: "gn01", password: "1001", label: "김교수", role: "member" }
```

예를 들어 성함을 바꾸려면:

```js
{ id: "gn01", password: "1001", label: "홍길동", role: "member" }
```

관리자 계정 예시:

```js
{ id: "admin", password: "admin1234", label: "관리자", role: "admin" }
```

## 4. 운영 순서

1. 교수님께 아이디와 초기 비밀번호를 안내합니다.
2. 첫 로그인 후 `비밀번호 변경`을 사용하도록 안내합니다.
3. 성함이나 계정을 바꿔야 하면 `js/main.js`를 수정합니다.
4. 수정 후 GitHub에 다시 올리면 실제 홈페이지에 반영됩니다.

## 5. 꼭 알아둘 점

- 이 방식은 `시험용`입니다.
- 공개 GitHub Pages 사이트에서는 보안이 강하지 않습니다.
- 브라우저 저장 방식이라, 컴퓨터를 바꾸면 비밀번호가 공유되지 않습니다.
- 실제 운영용이라면 나중에 Firebase 같은 정식 로그인 방식으로 바꾸는 것이 좋습니다.
